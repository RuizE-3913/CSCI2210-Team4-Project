In this stage, I worked on the Airline, Airport, and Staff classes. The following are the classes, methods, and reasoning we are applying to the airline management system:

class Airline {
  + airlineCode: String     // e.g. "AA", "UA", "BA"
  + name: String            // e.g. "American Airlines"
  + headquarters: String    // e.g. "Dallas, USA"
  + manageFleet()           // manage planes
  + operateFlight()         // run flights
}

class Airport {
  + airportCode: String         // "JFK", “LAX”
  + name: String                // Official airport name 
  + city: String                // e.g. "New York"
  + country: String             // e.g. "United States"
  + getGates()                  // Retrieve list of available gates / gate assignment info
}

class Staff {
  + staffId: String             // Unique employee identifier 
  + name: String                // Full name of the staff member
  + email: String               // Work email address
  + hireDate: Date              // Date when the employee was hired
  + department: String         // e.g. "Flight Operations", "Cabin Services", "Ground Handling"
  + role: String               // Pilot, CabinCrew, GroundStaff, Admin, etc.
  + status: String             // e.g. "Active", "On Leave", "Training", "Retired"
  + assignToFlight()           // Assign this staff member to a specific flight/crew
  + updateAvailability()       // Update availability status 
}
